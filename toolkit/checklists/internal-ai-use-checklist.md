# Internal AI Use Checklist

Use this checklist when enabling or expanding internal use of AI tools (including genAI).

This is a practical starting point and does not provide compliance assurance. See [../DISCLAIMER.md](../DISCLAIMER.md).

## 1. Tools and Access

- [ ] Tool is approved (or approved through procurement review)
- [ ] Access is limited to intended teams/users
- [ ] Admin settings reviewed (logging, data retention, training opt-out, sharing)
- [ ] Sharing controls reviewed (public links, external sharing, plugin access)
- [ ] A responsible owner is assigned for the tool configuration

## 2. Data Rules

- [ ] "Do not input" rules communicated (secrets, confidential info, sensitive personal data)
- [ ] Allowed data types documented
- [ ] Storage and retention understood (at least at a high level)
- [ ] Guidance exists for redaction/summarization before using AI
- [ ] Guidance exists for using internal vs external tools (what goes where)

## 3. Human Oversight

- [ ] Clear rule: humans remain responsible for decisions and outputs
- [ ] Review required for customer-facing communications
- [ ] Review required for high-impact decisions (HR, finance, legal, safety)
- [ ] No automated sending/acting is enabled by default (unless explicitly approved)

## 4. Training

- [ ] Staff completed AI literacy onboarding
- [ ] Training completion recorded
- [ ] Staff understand common failure modes (hallucinations, bias, automation bias)

## 7. Minimum Guardrails (Practical)

- [ ] Users are told to verify factual outputs before use
- [ ] Users are told not to treat outputs as professional advice to customers
- [ ] A path exists to report unsafe outputs or suspected data leakage immediately
