# Progress notes (local examples)

This folder is for **optional, local** readiness notes you generate while using the toolkit (for example after a self-assessment). They are **not** required for the repository and are **not** committed when they match the ignored filename pattern.

## What to do locally

1. Copy `EXAMPLE_PROGRESS_NOTE.md` to a new file or use your own naming outside the ignored pattern.
2. Or create files named `toolkit-progress-note-*.md` under a subfolder (for example `my-org/`); those names are listed in `.gitignore` so they stay on your machine only.

## In the repository

Only the example file and this README are tracked. Keep notes with confidential or client-specific content out of git.
