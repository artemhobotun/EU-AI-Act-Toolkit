# Sector pack: Vendor procurement and SaaS

## Typical SME use cases

- buying AI-enabled SaaS
- evaluating vendors
- contract renewal review
- admin and security review
- rollout approval

## Common tools

- AI-enabled SaaS products
- model APIs
- procurement questionnaires
- security review checklists

## Typical data involved

- vendor documentation
- contract terms
- security materials
- DPA documents
- procurement notes

## Key readiness questions

- What AI functionality is included?
- Is the system GPAI-based?
- What documentation is available?
- What data is used for training or inference?
- Are customer inputs used for training?
- What audit logs exist?
- What security controls exist?
- What support is provided for EU AI Act readiness?

## Basic risk screening notes

This is a practical starting point for procurement. If the vendor cannot explain training data, logs, security, or human oversight support, the use case should be escalated.

## Documentation to keep

- vendor questionnaire
- procurement approval
- contract or DPA reference
- security notes
- decision log
- incident log if needed

## Internal policy controls

- procurement sign-off route
- security and privacy review before activation
- vendor change review
- approved vendor list

## Human review expectations

Human review should happen before purchase approval and before any expanded deployment.

## When to escalate

- vendor documentation is missing or unclear
- customer inputs are used for training without clarity
- logs or transparency information are inadequate
- the tool will affect people, rights, or sensitive decisions
