# Templates

Use these templates as a practical starting point and adapt them to your sector, vendor, and internal process.

| Template | Format | Purpose | When to use |
|---|---|---|---|
| [ai-system-inventory.csv](ai-system-inventory.csv) | CSV | Track AI systems, owners, and review dates | When creating or refreshing the register |
| [ai-system-inventory.md](ai-system-inventory.md) | Markdown | Capture a fuller system profile | When you want narrative notes and evidence links |
| [ai-risk-screening-form.md](ai-risk-screening-form.md) | Markdown | First-pass screening and escalation notes | When reviewing a new or changed AI use case |
| [vendor-ai-questionnaire.md](vendor-ai-questionnaire.md) | Markdown | Practical supplier questions | Before buying or renewing a vendor tool |
| [ai-use-policy-template.md](ai-use-policy-template.md) | Markdown | Internal AI use rules | When publishing or refreshing policy |
| [ai-incident-log.md](ai-incident-log.md) | Markdown | Record issues and follow-up actions | After incidents or near misses |
| [employee-ai-literacy-record.md](employee-ai-literacy-record.md) | Markdown | Record training | When onboarding staff or refreshing training |
| [evidence-pack-index.csv](evidence-pack-index.csv) | CSV | Evidence pack index | When tracking documents across use cases |
| [maintenance-review-log.md](maintenance-review-log.md) | Markdown | Maintenance review record | When reviewing sources, wording, or releases |
| [vendor-ai-due-diligence-questionnaire.md](../vendor-pack/templates/vendor-ai-due-diligence-questionnaire.md) | Markdown | Vendor due diligence questions | Before buying or renewing an AI-enabled SaaS tool |
| [vendor-comparison-matrix.csv](../vendor-pack/templates/vendor-comparison-matrix.csv) | CSV | Compare vendors side by side | When evaluating multiple vendors |
| [vendor-risk-register.md](../vendor-pack/templates/vendor-risk-register.md) | Markdown | Track vendor risks | During review and escalation |
| [vendor-decision-record.md](../vendor-pack/templates/vendor-decision-record.md) | Markdown | Record vendor decisions | After evaluation |
