# 30-Minute AI Readiness Self-Assessment

## Purpose

Use this short assessment to get a quick first view of your AI readiness work. It is a starting point, not a final assessment.

## How to use

- Check the items that apply.
- Keep the wording practical.
- Escalate anything that may require deeper review.
- Store the completed copy internally.

## Section 1: AI tools we use

- [ ] We know which AI tools or AI-enabled features are in use.
- [ ] We have named an owner for each important tool.
- [ ] We know whether the tool is internal, vendor-hosted, or both.

## Section 2: People affected

- [ ] The tool does not affect people in a direct way.
- [ ] The tool may influence customers, workers, candidates, or users.
- [ ] The use case should be escalated because it affects people.

## Section 3: Data involved

- [ ] No personal data is involved.
- [ ] Personal data may be involved.
- [ ] Special category or sensitive data may be involved.

## Section 4: Vendor clarity

- [ ] We know what the vendor says the AI functionality does.
- [ ] We have asked whether customer inputs are used for training.
- [ ] We have requested documentation and security details.

## Section 5: Human review

- [ ] A human reviews important outputs before action is taken.
- [ ] Staff can challenge or correct the system.
- [ ] There is a fallback if the tool makes a poor output.

## Section 6: Documentation

- [ ] We have an inventory entry.
- [ ] We have a screening note.
- [ ] We have vendor questions or vendor answers.
- [ ] We know where evidence is stored internally.

## Section 7: Escalation triggers

- [ ] The tool affects employment, education, credit, healthcare, or essential services.
- [ ] The tool uses biometric or other sensitive methods.
- [ ] The tool works without meaningful human oversight.
- [ ] The tool uses personal data and the vendor cannot explain the setup.

## Preliminary result

- **Low attention**: few flags, simple internal use, and clear human review.
- **Medium attention**: some data, vendor questions, or workflow gaps that need follow-up.
- **High attention**: people-affecting, sensitive, or uncertain use cases that should be escalated.
