How to contribute safely

  • keep examples fictional or sample only
  • avoid personal or confidential data
  • use cautious wording for legal-sensitive topics
  • update source notes when legal wording changes

Good first contribution ideas

  • fix a broken link
  • improve a checklist
  • add a vendor question
  • refine a Pages card or navigation link

What not to share

Do not post confidential business information, personal data, contracts, security details, or incident data in public GitHub issues.
