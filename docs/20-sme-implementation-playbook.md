# SME Implementation Playbook

## Purpose

This playbook gives small teams a practical way to start, document, and maintain EU AI Act readiness work.

## Who should own this

- one accountable owner for the toolkit
- one operational owner for the evidence pack
- one review contact for legal, privacy, or security escalation

## 30-minute quick start

1. Read the disclaimer.
2. List the AI tools you already use.
3. Mark which tools affect people or use personal data.
4. Put the inventory in one place.
5. Pick one owner.

## 1-day setup

1. Complete the inventory.
2. Run one risk screening per tool.
3. Download or copy the vendor questionnaire.
4. Decide which tools need human review.
5. Store all records in a simple folder structure.

## 1-week setup

1. Write or update the internal AI use policy.
2. Record AI literacy onboarding.
3. Add an incident log.
4. Set up review dates.
5. Identify use cases that should be escalated.

## 4-week implementation plan

1. Week 1: inventory and ownership
2. Week 2: screening and escalation
3. Week 3: vendor review and procurement controls
4. Week 4: policy, training, and evidence pack setup

## Quarterly maintenance plan

- review the inventory
- review new or changed vendor tools
- check for new official guidance
- update source notes if assumptions change
- refresh training records where needed

## What to do before buying a new AI tool

- ask the vendor questions
- check logs, security, and training defaults
- record the decision
- escalate if the use case affects people or sensitive data

## What to do before launching an AI-enabled feature

- complete a screening form
- check whether people are affected
- confirm human oversight
- confirm privacy and vendor review
- write down the launch decision

## What to do after an incident

1. Log the issue.
2. Stop or limit use if needed.
3. Review impact and cause.
4. Capture corrective actions.
5. Decide whether the case should be escalated.

## Evidence pack checklist

- inventory
- screening forms
- vendor questionnaires
- policy
- literacy records
- incident log
- decision log
- source notes

## Escalation checklist

- people affected
- personal data involved
- sensitive context
- vendor uncertainty
- missing human review
- unclear legal or privacy implications
