# AI Literacy Guide (SME Baseline)

The EU AI Act introduces "AI literacy" expectations. This guide helps you set a practical baseline for staff who use or manage AI systems.

This is intentionally lightweight. You should tailor it to your actual tools and risks.

Record template: [../templates/employee-ai-literacy-record.md](../templates/employee-ai-literacy-record.md)

## Minimum Topics To Cover

1. What AI tools are approved in your org and for what purposes.
2. What data must never be entered into third-party AI tools (for example secrets, credentials, sensitive personal data) unless explicitly approved.
3. Common failure modes:
   - hallucinations / confident errors
   - bias or unfair outcomes
   - data leakage via prompts/uploads
   - automation bias (over-trusting outputs)
4. Human oversight:
   - what requires human review before action
   - how to escalate if an output looks unsafe or wrong
5. Incident awareness:
   - what counts as an "AI incident" internally
   - how to log and report it

## Suggested Training Format (Low Overhead)

- 30-45 minutes onboarding session for anyone who uses AI tools
- 15 minutes refresh every 6-12 months, or after major tool changes
- short "approved tools" page + internal policy link
